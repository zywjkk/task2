# 工程编译运行说明及核心思路总结
## 一、工程编译与运行说明
### 1. 环境依赖准备
本工程基于 Python 语言开发，核心依赖 OpenCV 和 NumPy 库，需先配置如下环境：

  - 基础环境：安装 Python 3.6 及以上版本（推荐 3.8~3.10 版本）；
  
  - 依赖库安装：打开终端 / 命令行，执行命令
 ```bash
 pip install opencv-python numpy
 ```
，完成 OpenCV 和 NumPy 库的安装。

### 2. 代码准备与路径调整
1. 将 main.py 文件保存到本地任意目录；

2. 修正图像路径：代码中 “if name == "main"” 部分，初始化 shibie 类时传入的参数是 “识别图像.pdf”,可替换为实际待识别的图像路径，比如 “test.jpg”“target.png” 等，示例修改为：analyzer = shibie ("test.jpg")。

### 3. 工程运行
打开终端 / 命令行，切换到 main.py 所在的目录；
执行运行命令：python main.py；
 运行后效果：
 - 终端会输出颜色、形状的统计结果；
 - 弹出可视化窗口，展示标注后的识别结果；
 - 按下键盘任意键可关闭可视化窗口，程序结束。

## 二、核心功能实现思路与理论基础
### 1. 整体架构设计
代码通过 shibie 类封装所有视觉识别逻辑，核心流程为：

     图像加载与预处理 → 颜色掩码提取与精细化 → 形状特征分析与分类 → 结果统计与可视化，实现 “颜色 + 形状” 的双维度目标识别与统计。
### 2. 各模块思路与理论基础
（1）图像加载与预处理（load_image 方法）

核心思路：

1. 读取图像并校验有效性，避免无效路径导致程序异常；

2. 按最大维度 1000 像素等比例缩放图像，平衡处理效率与识别精度；

3. 执行双边滤波操作，在去除图像噪声的同时保留边缘信息，为后续颜色分割做准备；

4. 将 OpenCV 默认的 BGR 色彩空间转换为 HSV 空间，降低光照变化对颜色识别的干扰。

理论基础：

1. 双边滤波：结合空间距离和像素值相似度的滤波算法，既能去除噪声，又能保留目标边缘；

2. HSV 颜色空间：将颜色拆分为色相、饱和度、明度三个维度，色相维度对颜色的区分度更高，且受光照（明度）影响更小，适合颜色分割任务。

（2）颜色掩码提取与精细化（get_refined_mask 方法）

核心思路：

1. 针对红色等跨 HSV 区间的颜色，设置多组阈值生成初始颜色掩码；
2. 执行形态学开运算，消除掩码中的小噪声点，优化轮廓质量；
3. 查找轮廓并填充内部（内部灌浆），解决反光导致的掩码 “破洞” 问题；
4. 通过距离变换提取目标的 “确定前景”，结合动态阈值筛选核心区域；
5. 运用分水岭算法，精细分割粘连的目标，得到独立的目标轮廓。
   
理论基础：

1.   形态学开运算：先腐蚀后膨胀的组合操作，可有效去除小的亮区域噪声；
2. 轮廓检测：基于图像边缘的连通域分析，提取目标的外接轮廓；
3. 距离变换：计算掩码中每个像素到最近背景像素的欧式距离，距离值越高代表越接近目标核心；
4. 分水岭算法：将图像视为拓扑地形，通过标记确定前景、背景，对未知区域 “注水” 实现粘连目标的分割。

（3）形状特征分析与分类（classify_shape 方法）

核心思路：

1. 提取轮廓的面积、凸包、最小外接圆 / 矩形等基础几何特征，计算圆拟合度、长宽比等关键指标；
2. 基于距离变换分割目标核心与边缘区域，计算亮度均值和方差，区分平面目标（如圆盘）与曲面目标（如球体、圆柱）；
3. 修剪目标表面花纹，修正外接矩形，避免花纹干扰形状判断；
4. 融合几何特征与 3D 光照特征，完成球、圆盘、正方体、长方体、圆柱的分类。
    
理论基础：
1. 凸包：包含所有轮廓点的最小凸多边形，反映目标的整体几何轮廓；
2. 最小外接圆 / 矩形：基于轮廓的几何拟合，圆拟合度用于判断圆形度，长宽比用于区分方形与矩形；
3. 图像矩：通过矩计算目标质心（公式：cx = m10/m00、cy = m01/m00），用于标注文字的位置定位；
4. 亮度统计：曲面目标因光影变化，核心与边缘的亮度差更大，整体亮度方差更高，可区分平 / 曲面目标。

（4）结果统计与可视化（analyze+show_result 方法）

核心思路：
1. 遍历红、绿、蓝三色，统计各颜色下的目标数量；
2. 遍历每个目标轮廓，分类统计形状数量；
3. 在图像上绘制目标轮廓，添加 “颜色 - 形状” 标签（带背景框保证可读性）；
4. 终端输出统计结果，通过可视化窗口展示标注后的图像。
    
理论基础：
1. 轮廓绘制：使用 LINE_AA 参数实现抗锯齿，提升轮廓绘制精度；
2. 文本绘制：通过 getTextSize 计算文本尺寸，适配绘制背景框，保证标注清晰可读。
